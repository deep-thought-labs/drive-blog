## 🔍 元のフローで検出されたエラー

ラウンド1のプロセス中、チェーンのローンチを継続する前に修正が必要な、初期フローに**根本的なエラー**が特定されました。

### 何が起こったか？

最初に提供されたベースジェネシスファイルには重大な問題がありました：**最初から含まれているべきアカウントと残高が事前に追加されていませんでした**。

**実行されたフロー（不正）:**
1. 空のベースジェネシスが提供された（アカウントなし）
2. 各参加者がローカルのジェネシスファイルに個別に自分のアカウントを追加した
3. 各参加者が編集したジェネシスに基づいてgentxを作成した
4. すべてのgentxをコンパイルしようとすると、元のベースジェネシスに最初からアカウントがなかったため、プロセスが失敗する

**従うべきだった正しいフロー:**
1. 参加者がチームに公開アカウントを提供する
2. チームがすべてのアカウントと残高でベースジェネシスを事前に埋める
3. 完全なベースジェネシスがすべての参加者に提供される
4. 参加者は完全なベースジェネシスに基づいてgentxのみを作成する
5. チームがすべてのgentxを正常にコンパイルする

### 修正計画

このエラーを修正するために、プロセスは2つのフェーズに分けられます：

**フェーズ1: 情報収集**
- 参加者が編集したジェネシスファイルを送信する（追加したアカウントを含む）
- チームがすべての個別ジェネシスファイルをコンパイルして、すべてのアカウントと残高を抽出する
- チームが最初からすべてのアカウントを含む新しいベースジェネシスv2を作成する

**フェーズ2: 修正されたプロセス**
- 新しいベースジェネシスv2がすべての参加者に提供される
- 参加者がベースジェネシスv2をダウンロードする（すでにすべてのアカウントを含む）
- 参加者がベースジェネシスv2に基づいて新しいgentxを作成する
- 参加者が新しいgentxを送信する
- チームがすべてのgentxを最終ジェネシスにコンパイルする

## 📋 フェーズ1: 編集したジェネシスファイルを送信

このフェーズでは、元のプロセス中に編集したジェネシスファイル（追加したアカウントを含む）を送信する必要があります。

### ステップ1: 編集したジェネシスファイルの場所を特定

編集したジェネシスファイルは、元のプロセス中に作業したのと同じ場所にあります：

**コンテナ内:**
```bash
~/.infinited/config/genesis.json
```

**ホストシステムから（コンテナの外）:**
```bash
services/node2-infinite-creative/persistent-data/config/genesis.json
```

またはサービスディレクトリから：
```bash
cd services/node2-infinite-creative
ls -la persistent-data/config/genesis.json
```

### ステップ2: 正しいファイルであることを確認

抽出して送信する前に、ファイルにアカウントが含まれていることを確認します：

**コンテナから:**
```bash
# コンテナにアクセス
cd services/node2-infinite-creative
./drive.sh exec infinite-creative bash

# キーをリストしてアドレスを取得
infinited keys list --keyring-backend file --home ~/.infinited

# アドレスがジェネシスに含まれていることを確認（YOUR_ADDRESSを実際のアドレスに置き換える）
cat ~/.infinited/config/genesis.json | jq '.app_state.bank.balances[] | select(.address == "YOUR_ADDRESS")'
```

**ホストから:**
```bash
# YOUR_ADDRESSを実際のアドレスに置き換える
cat services/node2-infinite-creative/persistent-data/config/genesis.json | jq '.app_state.bank.balances[] | select(.address == "YOUR_ADDRESS")'
```

正しい残高でアドレスが表示される場合、それが送信する必要があるファイルです。

### ステップ3: サーバーからジェネシスファイルを抽出

ジェネシスファイルはDockerの永続ボリュームに保存されているため、手動でコピーする必要なく、ホストシステムからアクセスできます。

**サーバーで直接作業している場合:**

1. **Creativeサービスディレクトリに移動:**
   ```bash
   cd services/node2-infinite-creative
   ```

2. **ファイル用のディレクトリを作成:**
   ```bash
   mkdir -p ~/genesis-correction
   ```

3. **ジェネシスファイルをコピー:**
   ```bash
   cp persistent-data/config/genesis.json ~/genesis-correction/
   ```

**リモートサーバーにいる場合**、`scp`を使用してファイルをローカルコンピューターにダウンロードする必要があります：

```bash
# ローカルコンピューターから
# <user>、<server>、パスを設定に応じて置き換える
scp <user>@<server>:/path/to/drive/services/node2-infinite-creative/persistent-data/config/genesis.json ~/genesis-correction/
```

**`scp`コマンドの説明:**
- `<user>`: サーバーにログインするために使用するユーザー名
- `<server>`: サーバーのIPアドレスまたはドメイン名を指します（例：`192.168.1.100`または`my-server.com`）
- コロン（`:`）の後のパスは、サーバー上のファイルへの完全なパスです
- `~/genesis-correction/`は、ローカルコンピューター上の宛先ディレクトリです

**⚠️ 重要:** このコマンドを実行すると、転送を実行するための認証情報または承認をシステムが要求する可能性が非常に高くなります。これらは、サーバーにログインするときに使用するのと同じ認証情報（パスワードまたはSSHキー）です。

**完全な例:** ユーザーが`ubuntu`で、サーバーのIPが`192.168.1.100`で、作業ディレクトリが`/home/ubuntu/drive`の場合：
```bash
# ローカルコンピューターから
mkdir -p ~/genesis-correction
scp ubuntu@192.168.1.100:/home/ubuntu/drive/services/node2-infinite-creative/persistent-data/config/genesis.json ~/genesis-correction/
```

### ステップ4: ジェネシスファイルを圧縮

ジェネシスファイルをローカルコンピューターに取得したら（サーバーで直接コピーしたか、`scp`でダウンロードしたかに関わらず）、識別を容易にするためにモニカーを含む名前で圧縮します：

1. **ファイルがあるディレクトリに移動:**
   ```bash
   cd ~/genesis-correction
   ```

2. **モニカーでファイルを圧縮:**
   ```bash
   tar -czf genesis-<your-moniker>-round-1-correction.tar.gz genesis.json
   
   # またはzipを使用
   zip genesis-<your-moniker>-round-1-correction.zip genesis.json
   ```

   **例:** モニカーが`my-validator`の場合：
   ```bash
   tar -czf genesis-my-validator-round-1-correction.tar.gz genesis.json
   ```

   **⚠️ 重要:**
   - 圧縮ファイルには識別を容易にするためにモニカーを含める必要があります
   - 圧縮ファイル内の元の`genesis.json`ファイルは元の名前を維持します

### ステップ5: ジェネシスファイルを送信

圧縮ファイルをTelegramで**Cypher Xenia**（ユーザー名**@XeniaCypher88**）に送信します。

1. Telegramを開く
2. ユーザー**@XeniaCypher88**（Cypher Xenia）を検索
3. 圧縮ファイルを送信（例：`genesis-<your-moniker>-round-1-correction.tar.gz`）
4. **メッセージに含める:** "ラウンド1修正用の編集されたジェネシスファイル - モニカー: [your-moniker]"

**⚠️ 重要:**
- `genesis.json`を含む圧縮ファイルのみを送信し、gentxを含む完全なジェネシスは送信しない
- 送信する前に正しいファイルを送信していることを確認
- ジェネシスファイルのバックアップコピーを保持

### ステップ6: 確認

ジェネシスファイルを送信すると、チームが受信して処理します。チームは受信したすべてのジェネシスファイルをコンパイルして、ベースジェネシスv2を作成します。

**厳格な時間制限はありません**が、チームが修正を進められるように、ファイルを**できるだけ早く**送信することが重要です。

## ❓ よくある質問

### 編集したジェネシスファイルを送信しないとどうなりますか？

編集したジェネシスファイルを送信しない場合、チームはベースジェネシスv2にアカウントを含めることができず、チェーンのローンチに参加できません。

### ジェネシスファイルを送信するための時間制限はありますか？

厳格な時間制限はありませんが、チームが修正を進めてベースジェネシスv2を作成できるように、**できるだけ早く**送信することが重要です。

### 送信する前にジェネシスファイルを変更できますか？

いいえ、元のgentxを作成するために使用したのと同じジェネシスファイルを正確に送信する必要があります。追加の変更は行わないでください。

### 編集したジェネシスファイルがもうない場合はどうなりますか？

編集したジェネシスファイルがもうない場合は、開発チームに連絡して代替ソリューションを見つけてください。

### ベースジェネシスv2はいつ利用可能になりますか？

ベースジェネシスv2は、チームが参加者からすべての個別ジェネシスファイルを受信して処理した後に利用可能になります。チームは準備ができたら通知します。

**👉 ベースジェネシスv2が利用可能になったら、次の手順を参照してください：**
**→ [Launch Round 1 - Gentxを再度作成](/ja/posts/launch-round-1-create-gentx-again/)**
