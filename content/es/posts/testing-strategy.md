Nos complace anunciar el inicio oficial de la **fase de testing** para Project 42. Esta fase es crucial para validar todas las funcionalidades, comandos y flujos esenciales de nuestra cadena de bloques **Infinite Improbability Drive** antes del lanzamiento oficial.

## Estrategia de Testing

### Dos Cadenas Oficiales

Durante esta fase, trabajaremos con **dos cadenas oficiales**:

#### Pre-Mainnet (Cadena Principal en Fase Beta)

- Esta es la versión "pre-mainnet" de la cadena principal
- Su existencia es necesaria para realizar pruebas técnicas requeridas
- Tendrá un número limitado de validadores para mantenerla operativa
- **Será reiniciada desde cero** cuando termine la fase de testing beta
- Después del reinicio, se lanzará la mainnet oficial

#### Testnet (Cadena de Pruebas)

- Esta será la cadena utilizada durante la fase inicial de testing
- Cualquiera puede configurar un nodo y convertirse en validador
- Se utilizará para probar, experimentar y asegurar que cualquier implementación funcione correctamente
- Los validadores que participen en esta fase serán **recompensados con tokens reales en la mainnet**

## ¿Cómo Participar?

Todos tienen la opción de apoyar el proyecto de dos maneras:

1. **Ejecutar un nodo** (tanto en testnet como en mainnet)
2. **Convertirse en validador**

### Proceso de Participación

1. **Configurar los nodos** y sincronizarlos
2. Recibir tokens iniciales [42] para crear validadores
3. Mantener ambas cadenas ejecutándose

## Características de las Rondas de Testing

### Testing Cíclico

Una característica clave es que **no será una sola ronda de testing**, sino que realizaremos **rondas cíclicas**:

- **Rondas semanales** y **cada 2 semanas**, dependiendo de la ronda
- **Cada jueves reiniciaremos la cadena** para comenzar una nueva ronda
- Esto nos permitirá recopilar métricas de cada ronda, compararlas y analizarlas

### Período de Unlocking Acelerado

- En la mainnet, el período definido para el desbloqueo gradual de tokens es de **42 años**
- Durante las rondas de testing, este período durará solo **1 semana**
- Esto nos permitirá experimentar el proceso de desbloqueo y liberación a un ritmo acelerado
- Podremos validar diversos comportamientos del sistema económico

## Próximos Pasos

En los próximos días, publicaremos:

- Instrucciones para instalar **Drive**, el cliente que gestionará los nodos en la cadena
- Guías para iniciar el nodo en la testnet
- Documentación detallada del proceso de configuración

Cada participante que configure su nodo recibirá tokens [42] para crear su validador y comenzar a validar.

## Tipos de Tests

Durante este período, se realizarán diversos tests para validar:

- Diferentes características de la cadena
- Comandos esenciales
- Flujos críticos del sistema
- Tests generales y grupales coordinados por el Lab
- Tests que involucrarán comandos de terminal
- Tests que utilizarán herramientas o plataformas específicas

## Actividades Planificadas

Durante las rondas de testing, realizaremos:

- Movimiento de tokens
- Votación en la DAO
- Simulación de escenarios "black swan"
- Y muchas otras actividades divertidas e improbables

## Importancia de esta Fase

Esta fase de testing es esencial porque:

- Nos permite validar todos los aspectos técnicos antes del lanzamiento oficial
- Identificamos y corregimos posibles problemas
- Aseguramos que el proceso de lanzamiento de la mainnet sea perfecto
- Creamos un historial documentado de cada ronda para análisis futuro
